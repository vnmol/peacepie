На основе самописного пакета peacepie, представляющего собой систему акторов с динамической загрузкой акторов из внешних пакетов,
строиться система универсального бэкенда, функциональность которого можно менять на ходу, загружая нужный набор акторов.
Взаимодействие акторов осуществляется через сообщения, записываемые в очереди соответствующих акторов.
Адресация осуществляется по именам акторов. Маршрутизацией занимается сам пакет peacepie.
В общем виде сообщение представляет собой словарь вида {"command": command, "body": body, "timeout": timeout, "recipient": recipient}.
В настоящее время разрабатывается актор реализующий веб-интерфейс для управления системой.
Частично функциональность актора уже реализована на основе связки FastAPI + JS + WS.
Текущая задача - разработать подсистему управления ролями, состоящую из html страницы и кода роутера для FastAPI.
Описание БД хранится в приложенном файле users.db.sql. 
Данные будут поступать от другого компонента в виде сообщений.
На странице html должна быть таблица из N рядов для просмотра и редактирования данных из БД.
Если в таблице БД больше чем N рядов, то html таблица должна оставаться заданного размера, а к ней должен добавляться скроллинг,
данные должны храниться только для видимых рядов, остальное должно подгружаться динамически при прокрутке.
Для поиска, выбора, редактирования ссылочных значений должно открываться модальное окно.
